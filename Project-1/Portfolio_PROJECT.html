// 01 — Overview

What is this project?

Sepsis kills ~11 million people annually worldwide. We built a full ML pipeline to detect it early from real ICU time-series data — addressing one of the hardest problems in clinical machine learning: severe class imbalance.

🩺

Clinical Problem

Sepsis is triggered by the body's dysregulated response to infection. Early detection is critical — every hour of delay increases mortality by 7%.

⚠️

Adversarial Challenge

Only 7.27% of patients develop sepsis. A model predicting "no sepsis" always achieves 92.7% accuracy — yet saves zero lives. We fix this.

📊

Dataset

1.55 million hourly ICU observations across 40,336 unique patients. 15 clinical features including vital signs, labs, and demographics.

🎯

Goal

Maximise recall on the minority (sepsis) class while maintaining overall model utility — optimising for AUC-ROC and F1, not raw accuracy.

// 02 — Methodology

Pipeline Overview

A reproducible, end-to-end pipeline from raw ICU data to evaluated classifier.

01

Data Loading & Aggregation

1.55M hourly rows aggregated to 40,336 patient-level records. For each vital sign (HR, O₂Sat, Temp, SBP, MAP, DBP, Resp) we compute mean, std, min, max — capturing both baseline and instability.

34 engineered features
02

Preprocessing

Median imputation fitted on training set only. Stratified 80/20 train-test split preserving the 7.27% class ratio. StandardScaler applied post-split to prevent leakage.

No data leakage
03

SMOTE Rebalancing

Applied to training set only. Synthetic minority samples generated by interpolating between existing sepsis-positive patients (k=5 neighbours). Training set: 2,346 → 29,922 positive samples.

Balanced training set
04

7-Model Training with CV

Logistic Regression, Decision Tree, Random Forest, SVM, KNN, Naive Bayes, and Gradient Boosting trained with stratified 2-fold CV. class_weight='balanced' applied where supported.

Stratified 2-fold CV
05

Hyperparameter Tuning

GridSearchCV on top 3 models (Logistic Regression, Random Forest, Gradient Boosting). Optimised for F1-macro to balance precision and recall across both classes.

GridSearchCV · F1-macro
06

Test Set Evaluation

Final evaluation on 8,068 held-out, real (imbalanced) patients. Metrics: AUC-ROC, F1-macro, F1-sepsis, precision, recall, and confusion matrix.

8,068 real patients
// 03 — The Problem

Minority Class Suppression

Before applying any mitigation, every model converged to predicting "no sepsis" for almost every patient — achieving high accuracy but detecting zero sepsis cases.

// Our Solution

Three-Layer Fix

  • 🔬

    SMOTE Oversampling

    Synthetic minority over-sampling in feature space (k=5) applied only to training data. Balances classes without duplicating real patients.

  • ⚖️

    Class-Weighted Loss

    class_weight='balanced' — penalises misclassifying sepsis patients more heavily during training.

  • 📈

    Recall-Oriented Metrics

    Evaluation uses AUC-ROC and F1-sepsis, not accuracy. A false negative (missed sepsis) is far more costly than a false positive.
